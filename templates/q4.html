Postal Codes

  • The Python part of your answer has to be written in the app.py file, in the proper place, as indicated in the TODO comments.
  • You will need to create a HTML template (file templates/codes.html), extending the exam template.

FedEx Postal Code Format
  1. Create a HTML template (file templates/codes.html), extending the exam template. In the contents block of this template, you should have
    • a drop down select field, with 5 options: Brazil, Canada, China, Thailand and United Kingdom
    • an input text field for the user to enter a postal code
    • an image of the selected country's flag, inside a link referring to the Google Maps page for that country (for example https://www.google.com/maps/place/Canada). The flag image should be an external image.
  2. The goal is to validate the postal code entered by the user according the postal code format for the selected country. The correct postal code formats are given in the table below. In the table N represents a digit, while A represents a letter. Multiple rows for a country (here, for UK) indicates that there are multiple valid postal code formats for that country, therefore your code will need to accept all possible valid formats for the country.
  3. When a country is selected in the list, the acceptable postal code format(s) for this country should be displayed somewhere in the page. The flag and the Google Maps link should match the selected country.
  4. There should be input validation on the postal code text field to make sure the postal code follows the proper format for the selected country. There should be some visual feedback when the postal code doesn't follow the proper format. Your code should be case-insensitive (it should accept a mix of lower and upper case letters), but it should replace all lower case letters with their uppercase equivalent after validation.
  5. Create an endpoint for your templates/codes.html template in the file app.py. The route should be /codes. Put your code where indicated in the TODO comments.
